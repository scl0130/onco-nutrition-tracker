Methodology and Design Direction

This project uses a clinical dashboard layout influenced by modern health experiences: low visual noise, strong metric hierarchy, and card-based daily guidance.

Core Principles

  • Status-first dashboard hierarchy
  • Clear risk and alerts before detailed logs
  • Progressive disclosure for advanced controls

Data and Privacy

  • All entries are stored in browser local storage
  • No account required
  • No backend patient data transmission in current static build
